# 简历定制与制作 Skill

## 概述

针对具体职位申请定制简历。不要向所有单位投递同一份通用简历；应突出与目标岗位最相关的经历，并匹配招聘人员和 ATS（申请人跟踪系统）使用的语言。默认交付基于内置 Word 模板制作的 `.docx` 成品，而不只是文字建议。

### 可以完成

- 分析职位描述并识别关键要求。
- 调整经历顺序，突出相关技能和成果。
- 提供 ATS 关键词优化建议。
- 用可核实的量化成果改善经历要点。
- 针对不同行业调整语气和表达。
- 将完整内容写入指定的内置模板并进行版面检查。

### 不能完成

- 编造用户没有的经历、学历、技能、证书、成果或数字。
- 保证用户一定获得工作、录取或面试。
- 用关键词包装代替真实资历。
- 因页数、版面或相关性原因擅自删除用户已经提供的有效内容。

## 语言规则

- 默认使用中文进行询问、分析、修改说明和简历写作。
- 公司英文名、软件、技术术语、证书原名和论文标题等专有名称可保留原文。
- 用户明确要求英文或中英双语时，再切换相应语言。

## 开始前必须确认

开始提取材料、改写内容或制作 DOCX 前，先确认：

1. 模板：`贵大蓝色版`、`蓝色版`、`黑色版`三选一。
2. 投递目标：公司、学校或单位，以及目标岗位、项目或申请类型；有完整 JD、招生公告或招聘公告时一并获取。

若两项均未提供，直接询问：

> 请先确认两项信息：
> 1. 请选择简历模板：贵大蓝色版、蓝色版、黑色版（三选一）。
> 2. 这份简历准备投递哪个公司、学校或单位的什么岗位或项目？如有 JD、招生公告或招聘公告，请一并发给我。

若只缺一项，只询问缺失项。用户暂未确定单位时，至少确认目标岗位、岗位类别或申请类型。除非用户明确说“默认”“任意”或“你决定”，否则不能代替用户选模板；获得授权后默认使用`蓝色版`。

## 内置模板

只允许使用以下内置文件：

| 用户选择 | 内置文件 |
|---|---|
| 贵大蓝色版 | `assets/templates/贵大蓝色版.docx` |
| 蓝色版 | `assets/templates/蓝色版.docx` |
| 黑色版 | `assets/templates/黑色版.docx` |

匹配模板名称时先匹配“贵大蓝色版”，再匹配“蓝色版”。用户同时提到多个模板时，要求其最终三选一。

### 模板硬性边界

- 每份新简历从用户选定的内置模板副本开始。
- 只修改副本，绝不覆盖内置模板。
- 不从空白文档重新设计，不用相似文档替代模板。
- 不使用 `D:\wyh2\简历` 中的外部模板制作成品；该目录只能作为用户资料来源。
- 修改已生成简历时继续沿用其对应内置模板；用户要求换模板时，从新模板重新生成。
- 保留模板页面尺寸、主题色、字体层级、表格、边框、形状、页眉页脚、段落间距、分栏和装饰元素。

## 使用流程

### 第 1 步：收集材料

获取：

1. 当前完整简历（粘贴文本或提供文件）。
2. 完整职位描述、招生公告或申请要求，不要只要摘要，因为精确关键词很重要。
3. 公司、学校或单位及目标岗位、项目或申请类型。
4. 其他背景，例如转行、内部转岗、工作空档、博士申请或特别关注事项。
5. 姓名、电话、邮箱、所在地、教育、工作、实习、科研、论文、项目、技能、证书、获奖和日期。
6. 已选模板。

只补问会阻止交付的关键信息，并尽量集中提问。

### 第 2 步：分析匹配关系

识别：

- 用人单位要求的关键技能和资历。
- 用户经历与要求的对应关系。
- 需要说明的差距和应重点强化的优势。
- 必备条件、优先条件、行业术语和高频关键词。

建立“岗位要求—用户证据”表，区分已有证据、可合理改写和确实缺失。将最相关且证据最强的内容前置，但不得删除其他已提供内容。

### 第 3 步：生成定制简历

提供：

- 优化后的完整简历。
- 主要修改及修改理由。
- 关键词匹配和申请建议。
- 基于所选内置模板制作的 DOCX 成品。

## 原始内容完整保留规则

- 用户消息、现有简历和附件中已经写出的有效内容都必须进入成品，包括论文题目、作者排序、期刊、分区、IF、专利名称、项目名称、获奖级别、课程、技能、职责、业绩和日期。
- 优化只能纠错、重组、前置重点和改善表达，不能用笼统概括替代并删除具体事实。即使增加总结，也必须保留支撑总结的具体条目。
- 不得因为控制页数、版面空间、相关性较弱或信息不完整而删除用户已提供的内容。可压缩措辞、合并重复表达和优化排版，但不能损失事实。
- 只有用户明确要求删除、隐藏、合并或精简某项时，才可以不写入该项。
- 写入前建立“原始内容—成品位置”核对清单，确保每项有效内容都有明确去向。

### 不完整信息处理

- 信息没有写完整时，保留所有已知部分，并在缺失字段原位填写 `xxx`。
- 不得留空、使用 `[待补充]`、删除整个条目或自行猜测。
- 示例：已提供论文题目、期刊和分区，但 IF 为空时，写成 `IF=xxx`；不得删除论文或只写“发表于相关期刊”。
- 示例：单位和职位已知但日期缺失时，写成 `单位 | 职位 | xxx—xxx`。

## 三页以内规则

- 所有成品必须控制在 3 页以内，包括求职简历、博士申请简历和学术 CV。
- 原始内容完整性优先，但必须通过结构重组、删除重复表达、缩短冗余句子、合理调整字号、行距、段距和模块布局，使成品不超过 3 页。
- 不得通过删除唯一事实、论文条目、项目、专利、获奖或其他有效内容来满足页数要求。
- 不使用影响可读性的极小字号、过密行距或明显拥挤的布局。
- 若完整内容确实无法在可读条件下放入 3 页，应停止交付并向用户说明冲突，请用户明确选择可删除或移至附件的内容；未经用户同意不得擅自删减。

## ATS 优化

多数大型单位使用 ATS 在人工查看前筛选简历。ATS 通常：

1. 扫描与职位描述匹配的关键词。
2. 根据关键词和条件匹配程度排序。
3. 将排名靠前的简历提交人工审核。

### 优化原则

| 应该做 | 不应该做 |
|---|---|
| 使用招聘公告中的精确关键词 | 使用 ATS 可能无法识别的花哨同义词 |
| 写出缩写全称并保留缩写 | 假设 ATS 理解所有缩写 |
| 使用标准模块标题 | 使用“我的旅程”等创意标题 |
| 提交 DOCX；用户要求时可另附 PDF | 提交不常见格式 |
| 在固定内置模板内保持清晰结构 | 为堆关键词破坏可读性 |

模板可能包含表格、分栏和图形，必须保留其结构；因此 ATS 优化应优先作用于文字、标题和关键词，不得破坏内置模板。

## 强动词分类

中文简历应优先选择准确、有证据支持的行动动词：

- 领导：指导、领导、管理、统筹、牵头、推动、开创、建立、启动。
- 成果：实现、交付、超额完成、提升、降低、改进、优化、精简。
- 技术：开发、设计、实现、架构、自动化、集成、迁移、部署、扩展。
- 沟通：协作、汇报、谈判、影响、倡导、撰写、记录、培训、指导、协调。

## 量化表达公式

把弱要点改为强要点：

**弱：**“负责销售。”

**强：**“通过新增 45 个企业客户，实现 250 万美元新增收入。”

**公式：** `[强动词] + [做了什么] + [可核实的量化结果] + [方法或背景]`

示例：

- “通过实施 CI/CD 流水线，将部署时间缩短 75%（从 4 小时降至 1 小时）。”
- “管理分布在 3 个时区的 12 人跨职能工程团队，按期交付产品。”
- “通过实施主动支持计划，将客户留存率提高 23%。”

只有原始材料能够证明数字时才使用；没有可靠数字时不得虚构。

## 行业化指导

### 技术 / 软件工程

应突出：

- 技术技能，包括语言、框架和工具。
- 项目经历；公开时可附 GitHub 链接。
- 带有影响指标的工作经历。
- JD 中提及的具体技术、Agile、Scrum、DevOps，以及用户量、事务量、数据规模等规模指标。

示例：

> “设计微服务迁移方案，将延迟降低 40%，并支持 10 倍流量增长。”

### 商业 / 咨询

应突出：

- 展示专业能力的职业概述。
- 关键成果。
- 教育经历，尤其适用于初级岗位。
- 收入、效率、ROI、Six Sigma、Lean、战略和客户或利益相关者管理。

示例：

> “领导战略项目，在 6 个业务单元实现 1500 万美元成本节约。”

### 市场 / 创意

应突出：

- 作品集链接。
- 带指标的营销活动成果。
- 工具和平台能力，例如 Google Ads、HubSpot。
- ROAS、CTR、转化率，以及 SEO、社交媒体和邮件等内容类型。

示例：

> “通过覆盖 5 个渠道的整合内容策略，使合格潜在客户增长 150%。”

### 医疗健康

应突出：

- 执照和认证。
- 临床经历。
- 合规和监管知识。
- RN、CNA、Epic、Cerner、HIPAA、Joint Commission 等相关术语。

### 金融

应突出：

- CFA、CPA、Series 7 等执照或资格。
- Excel、SQL、Python 等技术技能。
- 交易或项目经历、交易规模与类型、监管知识、DCF 和 LBO 等方法。

## 内容改写规则

- 使用“行动 + 方法 + 结果”结构。
- 保留可靠数字并突出；没有数字时使用准确的非量化结果。
- 统一日期、标点、人称和术语，经历通常按时间倒序排列。
- 应届生优先突出教育、项目、实习和岗位相关技能；有经验者优先突出最匹配的业绩。
- 加入关键词前确认用户经历真实支持该关键词，不做关键词堆砌。
- 不编造学历、单位、项目、技能、证书、职级、业绩和数字。

## 写入模板

1. 定位本 Skill 目录，读取用户选择的 `assets/templates` 模板。
2. 将模板复制为独立输出文件，在副本中替换示例文字和占位内容。
3. 优先使用 Microsoft Word 自动化处理复杂 DOCX；没有 Word 时，使用能保留 OOXML 结构的工具。
4. 优先精确替换和编辑既有段落，不删除无法识别的图形、形状或装饰对象。
5. 输出命名为 `姓名_目标岗位_模板名称.docx`；姓名缺失时使用 `xxx_目标岗位_模板名称.docx`。
6. 生成后用 Word 统计页数并检查视觉版面，确认不超过 3 页。

## 输出格式

分析说明采用以下结构；实际交付仍应包含 DOCX：

```markdown
## 简历分析

### 岗位要求匹配
| 要求 | 用户证据 | 差距 |
|---|---|---|
| [关键技能] | [对应经历] | ✅/⚠️ |

### 关键词优化
建议加入：[关键词列表]
已经具备：[关键词列表]

## 定制简历

[完整优化后的简历内容]

## 主要修改

### 概述部分
- [修改内容及理由]

### 经历部分
- [修改内容及理由]

### 技能部分
- [修改内容及理由]

## 其他建议

1. [求职信建议]
2. [LinkedIn 或其他平台建议]
3. [面试准备建议]
```

除非用户要求，不输出冗长逐句修改记录。完成后简要说明模板、投递单位与岗位、主要优化方向、页数和 DOCX 完整路径。

## 示例

### 示例 1：技术岗位

**用户：** 这是我的简历和某公司高级软件工程师岗位招聘信息。

**分析：**

1. 5 年以上 Python 经验：✅ 用户有 6 年。
2. AWS 能力：⚠️ 原文提及较少，需要在不编造的前提下展开。
3. 团队领导：✅ 管理过 3 名工程师。
4. 微服务：⚠️ 原文未提及，应询问，不得自行加入。

建议将 Python 前置，补充原材料中已有的 AWS 服务和团队影响，并确认是否有微服务经验。

### 示例 2：转行

**用户：** 我是一名教师，准备转向教学设计，如何重新表达经历？

| 教学经历 | 教学设计表达 |
|---|---|
| 编写教案 | 设计学习课程 |
| 评估学生进度 | 制定评估策略 |
| 适应不同学习者 | 创建无障碍、差异化内容 |
| 使用教育技术工具 | 具备 LMS 平台使用经验 |

只有用户确实使用过 Articulate、Adobe 等工具时，才可加入技能部分。

## 提高结果质量的建议

1. 提供完整职位描述，精确关键词比摘要更有用。
2. 提供完整简历，便于选择重点，同时确保全部原始内容进入成品。
3. 说明顾虑，例如工作空档、转行或博士申请。
4. 指明公司、学校或单位，不同组织有不同要求和文化。
5. 根据反馈持续迭代。

## 常见问题

**问：是否需要多份简历？**

答：建议保留一份包含全部信息的基础简历，再针对每次申请定制；不同岗位类别可保留 2—3 个版本。

**问：简历应该多长？**

答：所有成品均不得超过 3 页。通常 0—5 年经验以 1 页为目标，5—15 年以 1—2 页为目标，15 年以上、高管、博士申请或学术 CV 可使用 2—3 页，但仍必须遵守 3 页上限和内容完整规则。

**问：是否要包含每一份工作？**

答：用户已提供的工作都要保留。较早或相关性较低的经历可压缩为一行，但未经用户明确同意不得省略。

**问：个人项目是否应该加入？**

答：用户提供的个人项目应保留；与岗位相关时重点展示，尤其适用于技术岗位和转行者。

## 限制

- 无法独立核验用户经历，因此必须忠于原材料。
- 无法完全了解单位内部文化，用户仍需另行调查。
- ATS 系统各不相同，无法保证通过所有系统。
- 所有优化都受真实资历和用户提供证据约束。
- 高管岗位可建议专业简历服务，但仍应完成用户要求的模板成品。

## 成品检查

交付前逐项确认：

- 模板与用户选择一致，内置模板原文件未被修改。
- 公司、学校或单位、目标岗位、项目和 JD 匹配方向正确。
- 所有事实、数字和日期都能从用户材料找到依据。
- 已逐项对照原始材料，所有有效内容均已进入成品，没有被概括性语句替代或遗漏。
- 所有不完整字段均以 `xxx` 原位补齐，没有空白字段或 `[待补充]`。
- 模板示例文字、示例姓名和无关占位内容已清理。
- 成品不超过 3 页。
- 没有文字溢出、遮挡、重叠、异常换页、孤立标题或空白尾页。
- DOCX 能正常打开，输出路径明确。

## 最终交付

用中文简要告知：

1. 使用的模板。
2. 针对哪个公司、学校或单位，以及什么岗位或项目制作。
3. 主要优化方向。
4. 最终页数（必须不超过 3 页）。
5. DOCX 成品完整路径。
